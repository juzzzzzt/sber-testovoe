# Task 2: Агент для генерации клиентской презентации

Агент генерирует персональные презентации (`.pptx`) для корпоративных клиентов банка по запросу (например, «Подготовь презентацию о РЖД»). Использует архитектуру **RAG** с семантическим поиском через **OpenSearch** и генерацией контента через **GigaChat API**.

---

## Структура проекта

```
task2/
├── data/                          # Исходные данные
│   ├── clients.json               # Профили клиентов и финансовые риски
│   ├── products.json              # Справочник финансовых продуктов (IRS, FX Forward, и т.д.)
│   ├── internal_notes.json        # Внутренние экспертные заметки — бизнес-логика подбора продуктов
│   └── news.csv                   # База новостей из различных источников
│
├── src/                           # Исходный код
│   ├── config.py                  # Конфигурация: загрузка .env, пути к данным
│   ├── data_ingester.py           # Загрузка данных в OpenSearch + генерация эмбеддингов (sentence-transformers)
│   ├── rag_retriever.py           # RAG-поиск: профиль, заметки, продукты, новости (гибридный: точный + семантический)
│   ├── llm_client.py              # Работа с GigaChat API: генерация контента в 3 этапа (профиль, новости, продукты)
│   ├── pptx_builder.py            # Создание .pptx с корпоративным дизайном (зелёный стиль Сбера)
│   └── main.py                    # Точка входа: приём имени клиента → RAG → GigaChat → PPTX
│
├── output/                        # Сгенерированные презентации
├── .env                           # Переменные окружения (токен GigaChat)
├── requirements.txt               # Зависимости
└── README.md
```

---

## Как это работает

1. **Data Ingestion** — данные из `data/` загружаются в OpenSearch. Для текстов (профили, заметки, новости) генерируются векторные эмбеддинги через `paraphrase-multilingual-MiniLM-L12-v2`.
2. **RAG Retriever** — по имени клиента извлекает профиль, внутреннюю заметку (ключевой источник бизнес-логики), продукты и новости. Поиск гибридный: точный по имени + семантический по эмбеддингам.
3. **GigaChat Generation** — LLM получает контекст и формирует структуру презентации в JSON. Промпты жёстко ориентированы на внутреннюю заметку (какие продукты предлагать, а какие нет).
4. **PPTX Builder** — рендерит файл `.pptx` с единым стилем, заголовками и читаемым текстом.

## Запуск

### 1. Зависимости
```bash
pip install opensearch-py sentence-transformers gigachat python-pptx python-dotenv
```

### 2. OpenSearch
Скачать с opensearch.org, распаковать, в `config/opensearch.yml` добавить:
```yaml
plugins.security.disabled: true
network.host: 127.0.0.1
```
Удалить папку `plugins/opensearch-security/` (иначе не запустится). Запустить `bin/opensearch.bat` (Windows) или `bin/opensearch` (Mac/Linux). Проверить: http://localhost:9200

### 3. Токен GigaChat
Создать файл `.env` в корне `task2/`:
```env
GIGA_CREDENTIALS=ваш_токен
OS_HOST=localhost
OS_PORT=9200
```

### 4. Загрузка данных
```bash
cd src
python data_ingester.py
```

### 5. Генерация презентации
```bash
python main.py "РЖД"
python main.py "Аэрофлот"
```
Результат — в папке `output/`.

## Известные ограничения

**Галлюцинации при подборе новостей.** Несмотря на гибридный поиск (точный + семантический), GigaChat иногда «додумывает» новости или выбирает нерелевантные — особенно для клиентов с коротким или неоднозначным названием (например, «Магнит» может пересечься с медицинскими новостями про МРТ). Это типичная проблема LLM при работе с новостным контекстом: модель стремится заполнить слайд, даже если релевантных новостей мало. В продакшене это решается строгим post-filtering'ом по `client_name` и ручной модерацией.
